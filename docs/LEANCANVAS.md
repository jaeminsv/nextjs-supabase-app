# KAIST 실리콘밸리 동문회 이벤트 관리 MVP — Lean Canvas

## 1. Problem (문제)

**Top 3 문제:**

1. **이벤트 정보의 휘발성**: 카카오톡 단톡방에 공지가 묻히고, 나중에 찾기 어려움
2. **회비 수금/정산의 수동 추적**: Venmo/Zelle 개인 송금 후 누가 보냈는지 스프레드시트로 수동 확인 — 100명 규모 이벤트에서 가장 큰 시간 소모
3. **참가 관리의 분산**: 구글 폼으로 참가 신청 → 스프레드시트로 취합 → 카톡으로 변동 관리, 도구가 3개로 흩어져 있음

**기존 대안 (Existing Alternatives):**

- 카카오톡 단톡방 + 구글 스프레드시트 + 구글 폼 조합
- Eventbrite, Meetup 등 범용 이벤트 플랫폼 (동문회 맞춤 기능 부족, 회비 추적 없음)
- 수기 관리 (주최자의 기억과 메모에 의존)

---

## 2. Customer Segments (고객 세그먼트)

**타겟 사용자:**

- KAIST 실리콘밸리 동문 약 300명
- 실리콘밸리 거주, 스마트폰으로 주로 접근
- 동문 총회, 딸기파티, 가을 산행 등 정기/비정기 모임 참여

**Early Adopters (초기 수용자):**

- 동문회 운영진 (이벤트 주최 빈도가 높아 페인포인트가 가장 큼)
- 가족 동반 모임에 자주 참석하는 동문 (게스트 관리, 차등 회비 등 복잡한 케이스)
- 테크 업계 종사자 (새로운 도구 수용에 거부감이 낮음)

---

## 3. Unique Value Proposition (고유 가치 제안)

**한 줄 메시지:**

> "카톡에 링크 하나 던지면, 참가 신청부터 회비 정산까지 한 곳에서 끝."

**High-Level Concept (X for Y 비유):**

- "동문회를 위한 Eventbrite + Splitwise"
- 이벤트 관리(Eventbrite)와 회비 정산(Splitwise)을 하나로 합친 동문회 전용 플랫폼

---

## 4. Solution (솔루션)

| 문제             | 솔루션                                                                     |
| ---------------- | -------------------------------------------------------------------------- |
| 이벤트 정보 휘발 | 영구적인 이벤트 페이지 + 카카오톡 공유 링크 (정보가 앱에 영구 보존)        |
| 회비 수동 추적   | "납부했어요" 셀프 신고 + 주최자 확인/반려 대시보드 (납부 현황 한눈에 파악) |
| 참가 관리 분산   | RSVP + 동반자(성인/아동) 수 + 회비 자동 계산을 단일 페이지에서 처리        |

---

## 5. Channels (채널)

- **카카오톡 단톡방**: 기존 커뮤니케이션 채널 활용 — 이벤트 링크 공유로 자연스러운 유입
- **동문회 운영진 직접 안내**: 운영진이 먼저 사용하고, 이벤트 생성 시 자동으로 회원들에게 노출
- **입소문**: 300명 폐쇄형 커뮤니티 특성상 빠른 전파 가능

---

## 6. Revenue Streams (수익 모델)

**MVP 단계:**

- 비영리 동문회 내부 도구로 수익 모델 없음 (무료)

**향후 확장 가능성:**

- Stripe 결제 연동 시 트랜잭션 수수료 (v1.1+)
- 다른 동문회/커뮤니티 그룹에 SaaS로 확장 시 구독 모델
- 프리미엄 기능 (이벤트 사진 갤러리, 고급 분석 등)

---

## 7. Cost Structure (비용 구조)

**고정 비용:**

- Vercel 호스팅: 무료 플랜 (소규모 트래픽)
- Supabase: 무료 플랜 (500MB DB, 50K MAU)
- 도메인: 연 $10~15

**변동 비용:**

- Supabase 유료 전환 시: $25/월 (사용량 증가 시)
- Vercel Pro: $20/월 (트래픽 증가 시)

**총 예상 비용: MVP 단계에서는 거의 무료 ($0~15/월)**

## 8. Key Metrics (핵심 지표)

| 지표                 | 설명                                 | 목표 (출시 3개월)    |
| -------------------- | ------------------------------------ | -------------------- |
| **MAU**              | 월간 활성 사용자 수                  | 동문 300명 중 100명+ |
| **이벤트 생성 수**   | 월별 생성된 이벤트 수                | 월 2~3개             |
| **RSVP 응답률**      | 게시된 이벤트에 대한 RSVP 비율       | 80%+                 |
| **회비 추적 사용률** | 유료 이벤트에서 앱 내 납부 신고 비율 | 70%+                 |
| **주최자 시간 절약** | 이벤트 1건 운영에 소요되는 시간 감소 | 50% 이상 감소        |

---

## 9. Unfair Advantage (경쟁 우위)

- **기존 커뮤니티**: 300명의 활성 동문 네트워크가 이미 존재 — 콜드 스타트 문제 없음
- **높은 전환 장벽**: 폐쇄형 커뮤니티 특성상 한번 정착하면 다른 도구로 이동할 동기가 없음
- **도메인 지식**: 실제 동문회 운영 경험에서 나온 요구사항 — 범용 도구가 제공하지 못하는 맞춤 기능 (게스트 유형별 차등 회비, KAIST 학력 관리 등)
- **카카오톡 연동 전략**: 대체가 아닌 보완 — 기존 습관을 깨지 않으면서 점진적으로 의존도를 낮추는 접근

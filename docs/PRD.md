# KAIST 실리콘밸리 동문회 이벤트 관리 MVP PRD

## 핵심 정보

**목적**: 카카오톡 + 구글 스프레드시트에 흩어진 이벤트 관리와 회비 수기 추적을 단일 모바일 웹 앱으로 통합하여 운영 부담을 줄인다.

**사용자**: KAIST 실리콘밸리 동문 약 300명 — 주로 스마트폰으로 이벤트 참석 여부를 확인하고 회비를 납부하는 실리콘밸리 거주 동문.

---

## 사용자 여정

```
1. 로그인 페이지
   ↓ Google OAuth 로그인 버튼 클릭

2. 신규 사용자 여부 판별 (자동)
   ↓
   [첫 로그인] → 회원가입 페이지 (다단계 위저드)
   [기존 사용자] → 역할 판별
                   ↓
                   [pending] → 승인 대기 페이지 (다른 페이지 접근 불가)
                   [member / admin] → 대시보드 페이지

3. 회원가입 페이지 (다단계 위저드)
   Step 1: 필수 정보 입력 (이름, 연락처)
   Step 2: KAIST 학력 정보 입력 (선택)
   Step 3: 직장 정보 입력 (선택)
   ↓ 제출 → 승인 대기 페이지로 이동

4. 승인 대기 페이지
   → 관리자가 회원 관리 페이지에서 승인 처리
   → 승인 완료 시 다음 로그인부터 대시보드 접근 가능

5. 대시보드 페이지 (승인된 회원)
   ↓ 예정 이벤트 카드 클릭 → 이벤트 상세 페이지
   ↓ 이벤트 목록 메뉴 클릭 → 이벤트 목록 페이지

6. 이벤트 목록 페이지
   ↓ 이벤트 카드 클릭 → 이벤트 상세 페이지
   ↓ 이벤트 만들기 버튼 클릭 (관리자/주최자만 표시) → 이벤트 생성/수정 페이지

7. 이벤트 상세 페이지
   ↓ RSVP 버튼 클릭 → 참석 여부 + 동반자 수 입력 → 동일 페이지 갱신
   ↓ "납부했어요" 버튼 클릭 → 납부 방법 선택 → 동일 페이지 갱신 (확인 대기 상태)
   ↓ 주최자/관리자: 참석자 & 회비 현황 버튼 → 참석자 및 회비 관리 페이지
   ↓ 주최자/관리자: 이벤트 수정 버튼 → 이벤트 생성/수정 페이지

8. 이벤트 생성/수정 페이지 (주최자/관리자)
   ↓ 저장(초안) / 게시 → 이벤트 상세 페이지 또는 이벤트 목록 페이지

9. 참석자 및 회비 관리 페이지 (주최자/관리자)
   ↓ 납부 확인/반려 버튼 클릭 → 동일 페이지 갱신

10. 회원 관리 페이지 (관리자 전용)
    ↓ 승인/반려 버튼 클릭 → 동일 페이지 갱신

11. 내 프로필 페이지
    ↓ 수정 완료 → 동일 페이지로 복귀
```

| -------- | ------------------- | ------------------------------------------------------------------------------ | ---------------------------------------- | --------------------------------------------------------------- |
| **F001** | 이벤트 CRUD         | 이벤트 생성, 조회, 수정, 상태 변경 (draft → published → cancelled / completed) | 서비스의 핵심 객체 관리                  | 이벤트 목록 페이지, 이벤트 상세 페이지, 이벤트 생성/수정 페이지 |

| **F003** | 회비 납부 및 확인   | 회원 "납부했어요" 셀프 신고 + 주최자 확인/반려 처리 + 납부 현황 요약           | 현재 가장 큰 페인포인트인 수기 정산 대체 | 이벤트 상세 페이지, 참석자 및 회비 관리 페이지                  |
| **F004** | 참석자 및 회비 현황 | 이벤트별 참석자 목록과 납부 상태를 한 화면에서 조회 및 확인/반려 처리          | 주최자의 정산 업무 효율화                | 참석자 및 회비 관리 페이지                                      |
| **F005** | 이벤트 공유 링크    | 이벤트별 고유 URL 복사 버튼 (카카오톡 공유용)                                  | 카카오톡 → 앱 유입 경로이므로 필수       | 이벤트 상세 페이지                                              |

### 2. MVP 필수 지원 기능

| **F015** | 대시보드 요약     | 예정 이벤트 목록과 내 RSVP 상태 한눈에 표시                   | 앱 진입 후 즉시 현황 파악                                        | 대시보드 페이지                                     |

## 메뉴 구조

```
모바일 앱 내비게이션 (하단 탭 바 — 로그인 후 공통)
├── 홈 (대시보드)
│   └── 기능: F015 (예정 이벤트 + 내 RSVP 현황 요약)
├── 이벤트 목록
│   └── 기능: F001 (게시된 이벤트 전체 목록, 예정/지난 이벤트 토글)
└── 내 프로필
    └── 기능: F014 (개인 정보, 학력, 납부 수단 수정)

비로그인 사용자
└── 로그인 페이지 - F010 (Google OAuth)

pending 사용자 (로그인 후, 승인 전)
└── 승인 대기 페이지 (다른 모든 경로 접근 차단)

주최자/관리자 추가 접근
├── 이벤트 만들기 버튼 (이벤트 목록 페이지 내)
│   └── 기능: F001 (이벤트 생성/수정 페이지 진입)
└── 참석자 & 회비 관리 버튼 (이벤트 상세 페이지 내)
    └── 기능: F003, F004, F013 (참석자 및 회비 관리 페이지 진입)

관리자 전용 메뉴 (하단 탭 바 추가 항목)
└── 회원 관리
    └── 기능: F012 (pending 회원 승인/반려, 관리자 승격)
```

## 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `F010` | **접근:** 비인증 사용자 전용 (인증 시 대시보드로 리디렉션)

| 항목            | 내용                                                                                                                                                                           |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **역할**        | Google OAuth를 통한 진입점. 신규/기존 사용자를 구분하여 적절한 페이지로 분기                                                                                                   |
| **진입 경로**   | 비인증 상태에서 임의 URL 접근 시 자동 리디렉션, 또는 직접 접근                                                                                                                 |
| **사용자 행동** | "Google로 계속하기" 버튼 클릭 → Google OAuth 팝업/리디렉션 → 인증 완료 후 자동 분기                                                                                            |
| **주요 기능**   | • Google OAuth 로그인 버튼<br>• 첫 로그인 감지 → 회원가입 페이지로 이동<br>• 기존 회원이고 pending 상태 → 승인 대기 페이지로 이동<br>• 기존 승인 회원 → 대시보드 페이지로 이동 |
| **다음 이동**   | 첫 로그인 → 회원가입 페이지, pending 회원 → 승인 대기 페이지, 승인 회원 → 대시보드 페이지                                                                                      |

---

### 회원가입 페이지

> **구현 기능:** `F011` | **접근:** 첫 Google 로그인 직후에만 진입 가능

| 항목            | 내용                                                                                                                                                                                                                                                                                                                                  |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 신규 동문 정보를 3단계 위저드로 수집하여 승인 요청 제출                                                                                                                                                                                                                                                                               |
| **진입 경로**   | 로그인 페이지에서 첫 로그인 감지 시 자동 이동                                                                                                                                                                                                                                                                                         |
| **사용자 행동** | Step 1 필수 정보 입력 → Step 2 KAIST 학력 입력(선택) → Step 3 직장 정보 입력(선택) → 제출 버튼 클릭                                                                                                                                                                                                                                   |
| **주요 기능**   | • **Step 1** 필수: 이름(full_name), 표시 이름(display_name), 연락처(phone)<br>• **Step 2** 선택: 학사/석사/박사 졸업 연도 및 전공, 석·박사 통합과정 여부<br>• **Step 3** 선택: 회사명, 직책, 납부 수단(Venmo/Zelle 핸들)<br>• 단계별 유효성 검사 (React Hook Form + Zod)<br>• 이전/다음 단계 이동 버튼<br>• **"승인 요청 제출"** 버튼 |
| **다음 이동**   | 제출 완료 → 승인 대기 페이지                                                                                                                                                                                                                                                                                                          |

---

### 승인 대기 페이지

> **구현 기능:** (없음 — 인터페이스 전용 페이지) | **접근:** role = pending 인 사용자 전용

| 항목            | 내용                                                                                                                                                                      |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | pending 상태 회원이 관리자 승인을 기다리는 동안 표시되는 안내 페이지                                                                                                      |
| **진입 경로**   | 회원가입 페이지 제출 완료 후 자동 이동, 또는 pending 상태로 로그인 시 자동 리디렉션                                                                                       |
| **사용자 행동** | 승인 안내 메시지 확인. 다른 액션 없음                                                                                                                                     |
| **주요 기능**   | • "관리자 승인을 기다리고 있습니다" 안내 메시지<br>• 관리자 연락 방법 안내 (카카오톡 등)<br>• 로그아웃 버튼<br>• 모든 다른 페이지로의 접근 차단 (proxy에서 리디렉션 처리) |
| **다음 이동**   | 관리자 승인 후 다음 로그인 시 대시보드 페이지로 이동                                                                                                                      |

---

### 대시보드 페이지

> **구현 기능:** `F015` | **접근:** role = member / admin, 로그인 필요

| 항목            | 내용                                                                                                                                                                                                                             |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 앱의 홈 화면. 예정 이벤트와 내 RSVP 현황을 한눈에 파악                                                                                                                                                                           |
| **진입 경로**   | 로그인 후 기본 진입 페이지, 하단 탭 바 "홈" 선택                                                                                                                                                                                 |
| **사용자 행동** | 예정 이벤트 카드를 훑어보고, 관심 이벤트 탭 → 이벤트 상세 페이지로 이동                                                                                                                                                          |
| **주요 기능**   | • 예정 이벤트 목록 (최대 3~5개, 날짜 순 정렬)<br>• 각 이벤트 카드: 제목, 날짜, 장소, 내 RSVP 상태 배지 (참석/미정/미응답)<br>• 내 미납부 이벤트 알림 배너 ("납부 대기 이벤트 N개")<br>• 이벤트 카드 탭 → 이벤트 상세 페이지 이동 |
| **다음 이동**   | 이벤트 카드 탭 → 이벤트 상세 페이지                                                                                                                                                                                              |

### 이벤트 목록 페이지

> **구현 기능:** `F001` | **접근:** role = member / admin, 로그인 필요

| 항목            | 내용                                                                                                                                                                                                                              |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 게시된 모든 이벤트를 탐색하는 목록 뷰                                                                                                                                                                                             |
| **진입 경로**   | 하단 탭 바 "이벤트 목록" 선택, 대시보드에서 "전체 보기" 링크                                                                                                                                                                      |
| **사용자 행동** | 예정/지난 이벤트 토글 전환, 이벤트 카드 탭으로 상세 진입, 관리자/주최자는 이벤트 만들기 버튼 클릭                                                                                                                                 |
| **주요 기능**   | • 예정 이벤트 / 지난 이벤트 토글 필터<br>• 이벤트 카드: 제목, 날짜, 장소, 참석 인원/정원, 내 RSVP 상태<br>• 이벤트 상태 배지 (published / cancelled / completed)<br>• **"이벤트 만들기"** 버튼 (관리자/주최자 권한 보유자만 표시) |
| **다음 이동**   | 이벤트 카드 탭 → 이벤트 상세 페이지, 이벤트 만들기 버튼 → 이벤트 생성/수정 페이지                                                                                                                                                 |

## 핵심 비즈니스 규칙

- **인원 계산**: 총 참석 인원 = 1(본인) + adult_guest_count + child_guest_count
- **회비 자동 계산**: 총 회비 = fee_amount + (adult_guest_count × adult_guest_fee) + (child_guest_count × child_guest_fee)
- **RSVP 변경 제한**: rsvp_deadline 이후 (없으면 start_at 이후) 변경 불가
- **납부 재신고**: rejected 상태인 경우 새 납부 레코드 신고 가능. (event_id, user_id) 조합에 pending 또는 confirmed 상태의 레코드는 1개만 허용 (애플리케이션 로직으로 강제)
- **이벤트 삭제 금지**: 물리적 삭제 없음. cancelled 상태로 변경
- **표준 시간대**: 모든 이벤트 일시는 Pacific Time (PT) 기준으로 표시
- **주최자 역할**: 프로필 레벨 역할이 아닌 이벤트별 권한 (event_organizers 테이블 기준)
- **다중 관리자**: 복수의 관리자가 동시에 회원 승인 및 주최자 위임 가능
